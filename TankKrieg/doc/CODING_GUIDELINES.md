# CODING GUIDELINES — TankKrieg

---

## 1. Naming

- Classes: PascalCase
- Functions: camelCase
- Variables: camelCase
- Constants: UPPER_CASE

---

## 2. File Structure

- One class per file (recommended)
- Header (.h) + Source (.cpp)

---

## 3. Comments

- Explain WHY, not WHAT
- Avoid obvious comments

---

## 4. Functions

- Small and focused
- One responsibility

---

## 5. Classes

- Must have a clear purpose
- Avoid large classes

---

## 6. Includes

- Minimize dependencies
- Use forward declarations when possible

---

## 7. Error Handling

- Fail early
- Avoid silent failures

---

## 8. Formatting

- Consistent indentation
- Clean spacing
